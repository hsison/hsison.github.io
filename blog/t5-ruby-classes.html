Sass, A$%, and Classes

Before diving deep into classes there are a few things you should know.
Ruby is an object-oriented language meaning, almost everything appears in Ruby as an object. This means every string, number, and true/false values are ALL OBJECTS.
Even a class itself is an object that is an instance of the Class class.

When To Use Classes

You use a class when you want to specify the form of an object. Classes combine data and methods into one nice little organized package.

Why Use A Class?

Although a single block of code is great for minor projects in programming, you need more structure. Ruby overall gives you a structured alternative to other programming languages.

How to use classes?

Throughout this tutorial you will learn how to create classes and use them. Roll up your sleeves and let's cook up some classes.
When creating a class you want to start with the keyword class followed by the name of your choosing. That's right, you can name your class anything! Just make sure you name it something reasonable incase other programmers want to read your code.
Also make sure the first letter of your class name is Capitalized.
You end your class using the keyword end. Any code in between the class and end keywords will be considered data members of that class.

What Does A Simple Class Look Like?

class Yourmom
def class members
end
end

What Does An Actual Class Look Like?

class Store
def initialize(name,products,workers)
@name = name
@products = products
@workers = workers
end
def description
return "I visited #{@name} and I bought #{@products}."
end
end
costco = Store.new("Costco","printers", "100")
costco.description
This class is creating a very broad class called Store.Since everything we do in programming is somewhat hypothetical, we're going to think like a computer.
Every time we create a new instance of the class Store the computer is literally going to think it's creating a store. Every store has a name, products and workers, these variables are accessible to everything within the store class because they are instance variables.
If you want something specific to only your store you can create what's called "local variables" within one single method.

Think About Why They Are Called Classes

The definition of classification is the action or process of classifying something according to shared qualities or characteristics.
The most important part of this definition is "SHARED QUALITIES OR CHARACTERISTICS".
This means when were creating classes, ever object within that class is going to have shared qualities or characteristics.
