My Experience Learning JavaScript

Learning JavaScript was quite interesting to me. Ruby, being the first language I had ever learned, was really difficult to learn. JavaScript, being the second language I have learned, was both weird, exciting, and uncomfortable for me. Ruby is very English-like and uses very simple syntax, while JavaScript uses more detailed syntax. Overall, it was a great learning experience and I'm so excited to further my knowledge of JavaScript and build some awesome stuff!

Ruby Classes vs. JavaScript Constructor Functions

In Ruby, a class is used to create an object. In JavaScript, a constructor function is used to create an object. Let me state the obvious. They both do the same thing in each language, but obviously there are going to be a few differences in syntax, creating, etc.

Creating an object in Ruby:

Ruby objects begin with the keyword class and the class name must have an uppercase first letter. Like so:

class Cool
end
That would create an empty class named Cool. An entire class would be written like this:
class Cool
def initialize(name,address,job)
@name = name
@products = address
@workers = job
end
end
Notice at the end of these classes I used the keyword end. Without the end keyword Ruby doesn't know if you're done yet.

Creating objects in JavaScript

JavaScript objects uses the keywords var and new. If I were to create an object in JavaScript it would look like this:

var hannah = new Object();
That would create an empty object named hannah.

If I wanted to add values into that object it would look like this:
var hannah = new Object();
hannah.name = "Hannah Smith";
hannah.age = 22;
Using the dot notation I added a name attribute that is assigned to the value "Hannah Smith". Also, I added an age attribute that is assigned to the value 22. Notice how I put a semi colon at the end? That is how JavaScript knows that line of code is complete.
